# Working with SARIF Reports

SARIF (Static Analysis Results Interchange Format) is a JSON-based format for sharing static analysis results. This guide explains how to effectively work with SARIF files generated by CodeQL.

## What is SARIF?

SARIF (Static Analysis Results Interchange Format) is a standard format (OASIS standard) for representing static analysis results. Key features:

- JSON-based format
- Highly structured with rich metadata
- Contains code locations, severity, and remediation advice
- Supports result categorization and filtering
- Designed for tool interoperability

## Viewing SARIF Reports

There are several ways to view SARIF reports effectively:

### 1. GitHub Security Tab

If you're using GitHub Advanced Security, SARIF results appear directly in the Security tab of your repository as "Code scanning alerts."

Benefits:
- Integrated with GitHub's UI
- Tracks alert status (open, fixed, dismissed)
- Shows code context and data flow paths
- Provides security severity scoring

### 2. VS Code with SARIF Viewer Extension

The [SARIF Viewer for VS Code](https://marketplace.visualstudio.com/items?itemName=MS-SarifVSCode.sarif-viewer) extension is an excellent way to analyze results locally:

1. Install the extension
2. Open your workspace in VS Code
3. Open the SARIF file or use the "SARIF: Open SARIF File" command
4. Browse results in the "SARIF Results" panel

Features:
- Interactive result navigation
- Code highlighting
- Data flow visualization
- Result filtering
- Customizable views

### 3. Web-Based SARIF Viewers

Several web-based tools exist for viewing SARIF:

- [SARIF Web Viewer](https://microsoft.github.io/sarif-web-component/)
- [CodeQL SARIF Viewer](https://lgtm.com/query/help/sarif-viewer)

These tools allow you to upload a SARIF file and explore results without installing additional software.

### 4. Command Line Viewing

For quick analysis, you can:

1. Convert SARIF to CSV:
   ```bash
   codeql bqrs decode --format=csv results.bqrs > results.csv
   ```

2. Use standard tools like `jq` to query SARIF directly:
   ```bash
   # Count results by severity
   jq '.runs[].results | group_by(.properties.security_severity) | map({severity: .[0].properties.security_severity, count: length})' results.sarif
   ```

## Understanding SARIF Structure

A SARIF file typically contains:

```json
{
  "$schema": "https://json.schemastore.org/sarif-2.1.0.json",
  "version": "2.1.0",
  "runs": [
    {
      "tool": {
        "driver": {
          "name": "CodeQL",
          "semanticVersion": "2.11.2",
          "rules": [...]
        }
      },
      "results": [
        {
          "ruleId": "js/eval-use",
          "message": {
            "text": "Avoid using eval or Function constructor..."
          },
          "locations": [...],
          "partialFingerprints": {...},
          "properties": {
            "security_severity": "7.0"
          }
        }
      ]
    }
  ]
}
```

Key sections:
- `tool`: Information about the analysis tool
- `rules`: Metadata about the rules/queries run
- `results`: The actual findings, with locations and messages
- `locations`: Where in the code the issue was found
- `properties`: Additional metadata about the result

## Formatting SARIF Reports for Presentations

When using SARIF reports in training or presentations:

1. **Use visual tools**: VS Code's SARIF viewer provides great visualizations
2. **Focus on key findings**: Filter to show only high-severity issues
3. **Show the data flow**: For path-problem queries, show the flow from source to sink
4. **Include code snippets**: Show the relevant code with the issue
5. **Explain remediation**: Always pair findings with remediation advice

## Converting SARIF to Other Formats

SARIF can be converted to other formats for easier consumption:

- **CSV** (good for spreadsheets and data analysis):
  ```bash
  codeql bqrs decode --format=csv results.bqrs > results.csv
  ```

- **HTML** (for standalone reports):
  ```bash
  # Using custom tools or scripts to generate HTML reports
  # Example: sarif-to-html results.sarif > report.html
  ```

- **Markdown** (for documentation):
  ```bash
  # Using custom tools or scripts
  # Example: sarif-to-md results.sarif > findings.md
  ```

## Integrating SARIF in CI/CD

SARIF reports are designed to integrate with CI/CD pipelines:

1. **GitHub Actions**: Automatically upload SARIF with `github/codeql-action/upload-sarif`
2. **Azure DevOps**: Use the SARIF SAST Scans Tab extension
3. **Jenkins**: Use plugins like Static Analysis Utilities
4. **Other CI Systems**: Upload results to a SARIF viewer or convert to formats compatible with your CI system

## Best Practices for SARIF Workflow

1. **Automate analysis**: Run CodeQL in CI/CD pipelines
2. **Save historical data**: Retain SARIF reports for trend analysis
3. **Review regularly**: Schedule regular reviews of findings
4. **Triage effectively**: Use GitHub Security's triage features
5. **Track remediation**: Monitor fix progress over time
